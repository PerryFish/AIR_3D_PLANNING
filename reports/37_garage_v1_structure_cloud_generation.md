# Garage V1 Structure Cloud Generation

## Source

Preferred source:

```text
/home/nuaa/ZHY/TARE_V1/src/autonomous_exploration_development_environment/src/vehicle_simulator/mesh/garage/preview/pointcloud.ply
```

This file is the TARE vehicle simulator garage preview point cloud. It contains `4,617,947` binary little-endian XYZ vertices.

## Generated AIR Files

Generated by:

```bash
scripts/generate_garage_structure_cloud_from_mesh.py --source tare --target-points 160000
```

Outputs:

```text
src/virtual_env/garage_v1/maps/garage_structure_from_tare.xyz
src/virtual_env/garage_v1/maps/garage_structure_from_tare.pcd
```

The generated cloud has `164,927` points.

## Transform

The script applies the same garage include translation used by Gazebo:

```text
(-23.817, -46.018, 0.073)
```

No scale or rotation is applied, matching `models/garage/model.sdf`.

Generated bounds:

```text
x [-63.881, 63.877]
y [-71.383, 71.384]
z [-0.037, 20.969]
```

These match the DAE/Gazebo model bounds within the expected downsampling tolerance.

## Runtime Publisher

`garage_structure_cloud_node` now loads the preferred cloud in this order:

1. `garage_structure_from_tare.pcd`
2. `garage_structure_from_tare.xyz`
3. `garage_structure_from_mesh.pcd`
4. `garage_structure_from_mesh.xyz`
5. generated wall proxy fallback

Published topics:

```text
/exploration/garage_structure_cloud
/exploration/static_garage_structure_cloud
```

QoS is transient local so RViz can receive the static cloud after node startup.

## Observed Structure

`sensor_mapping_node.py` now publishes:

```text
/exploration/observed_structure_cloud
```

This is online observed occupied structure from the simulated local sensor map. It is intentionally separate from the full static garage reference cloud.
